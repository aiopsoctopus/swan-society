TRIAGE SCENARIOS — pre-decided. read before Hr 1.
The principle: a smaller app that works end to end is a stronger submission than a bigger app with one visible broken state. Judges score "fully functional, beginning-to-end experience" — breadth does not compensate for broken.
SCENARIO A — TARGET

Full loop ships

Dashboard → AI book suggestions → voting → RSVP → AI discussion questions → pricing. All wired to Supabase. All functional on the deployed URL, cold, on a different device.

This is the plan. Aim here. Everything below is triage, not strategy.
SCENARIO B

Book suggestion edge function broken

⏱ Invoke when: stuck >30 min on book suggestion edge function past Hr 5
CUT: AI book suggestions. Replace with manual "host picks 3 books" flow.
KEEP: voting, RSVP, discussion questions. Loop still works end to end.

Pitch: "AI assists at the most important touchpoint — discussion questions. Book selection is host-driven in the MVP."

Prompt: "Remove the AI book suggestion edge function. Replace with simple host-entry: host types up to 3 titles/authors manually. Each card still has vote button + tally bar. Style consistently."

SCENARIO C — LAST RESORT

Discussion question generator broken

DO NOT pivot easily. This is your differentiator. Exhaust in order:

1. Simplify Groq prompt — strip criteria bullets to just: "Generate 6 thoughtful discussion questions for a book club reading {title} by {author}. Numbered list only."
2. Increase edge function timeout to 15s. Groq is fast — if it's timing out, it's the function setup, not the model.
3. Confirm GROQ_API_KEY is in Cloud → Secrets (not env vars). Most common gotcha.
LAST RESORT (past Hr 11 only): cached fallback + Regenerate cycles questions in random order. Remove spinner. It looks live. Never cut the screen entirely.
SCENARIO D

Something breaks after deploy

Order: Revert → De-emphasize → Remove → Seed around

Revert via Lovable version history — always first, takes 2 min.

De-emphasize: grey out / remove hover state ("Make [X] appear visually disabled without removing from layout").

Remove: "Remove [X] entirely and update navigation so nothing links to it."

Seed around: prevent broken state from being the default view.

SCENARIO E

Behind — nice-to-haves never happen

⏱ Invoke when: Hr 22 arrives and Prompts 12/13/14 were never sent
SKIP: Prompts 12, 13, 14 (affiliate links, host package, rotation view)
DO: Prompt 15 (polish) → seed demo data → deploy → cold judge test. This beats any nice-to-have.

CREATIVITY

Novel approach. Distinctive elements. Not a ChatGPT wrapper.
